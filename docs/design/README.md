# 設計ファイル

このディレクトリには、Claudeが作成した設計ファイルを配置します。

## 設計ファイルの目的
- 実装前に仕様を明確化
- AI（Codex/Cursor）への指示として利用
- レビュー・振り返りの材料

## ファイル命名規則
- kebab-case
- 機能名を含める
- 例: `oauth-flow.md`, `db-schema.md`, `post-generation.md`

## 設計ファイルに含めるべき内容

### 1. 概要
- 何を実装するか
- なぜ必要か
- 実装の優先度

### 2. 仕様
- 入出力
- データ構造
- API仕様（エンドポイント、メソッド、レスポンス）
- UI仕様（画面遷移、コンポーネント構成）

### 3. 制約・前提
- 技術的制約
- セキュリティ要件
- パフォーマンス要件

### 4. エラーハンドリング
- 想定されるエラー
- エラー時の挙動
- ユーザーへのフィードバック

### 5. テスト観点
- 何をテストするか
- テストケース例

## 設計ファイルのテンプレート

```markdown
# [機能名]

## 概要
[何を実装するか、なぜ必要か]

## 仕様

### 入出力
- 入力:
- 出力:

### データ構造
```typescript
type Example = {
  // 型定義
};
```

### API仕様（該当する場合）
- エンドポイント:
- メソッド:
- リクエスト:
- レスポンス:

### UI仕様（該当する場合）
- コンポーネント構成
- 画面遷移
- ユーザー操作フロー

## 制約・前提
-
-

## エラーハンドリング
| エラー | 原因 | 対応 |
|--------|------|------|
|        |      |      |

## セキュリティ要件
-
-

## テスト観点
- [ ]
- [ ]

## 実装時の注意点
-
-

## 参考
- [関連ドキュメントへのリンク]
```
