# Definition of Done Checklist

Use this checklist before reporting completion.

- [ ] Changed code satisfies all in-scope tasks.
- [ ] Out-of-scope items were not implemented.
- [ ] Acceptance criteria are explicitly mapped to implemented changes.
- [ ] Input validation and auth/authorization checks are present where needed.
- [ ] Error handling exists for expected failure paths.
- [ ] Type safety is preserved (`any` not introduced without reason).
- [ ] Tests/checks relevant to the change were run, or gaps were reported.
- [ ] Final report includes file paths and any unresolved risks.
